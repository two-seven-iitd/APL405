### 5. First run produced non-physical results → root-caused + fixed

First run's numbers made no sense:
- Physics-Only baseline varied between 23.42% (noise expt) and 5.12% (sparsity expt) —
  same model, should be identical.
- Data-Only sparsity was non-monotonic: 34% → 22% → 37% → 5% across sensor counts.
- Hybrid at 10 sensors (45%) was worse than Data-Only (22%) — impossible with
  physics regularization.

**Root cause**: the project had no actual weight seeding anywhere.
- `train.py` accepts `seed` but only uses it for `get_sensor_data(seed=seed)`.
- `torch.manual_seed` was never called, so Xavier weight init was truly random.
- `train_physics_only` had no `seed` parameter at all.
- `main.py`'s Physics-Only = 3.58% was a lucky init, not a reproducible result.
- Each experiment run got a different random network, drowning out the actual
  noise/sparsity signal.

**Fixes in experiments.py**:
1. Load Physics-Only from saved weights (`results/cantilever/models/physics_w.pth`)
   instead of retraining — frozen stable baseline.
2. Added `_seed_all(seed)` helper: `torch.manual_seed`, `torch.cuda.manual_seed_all`,
   `np.random.seed` — applied before each training run.
3. Bumped seeds from 3 → 5: SEEDS = [42, 43, 44, 45, 46].
4. Added error bars (mean ± std) to both plots and summary tables.

**Note**: main.py itself still isn't seeded, by design. Its saved `physics_w.pth`
is the frozen baseline that experiments.py loads. Re-running main.py with seeding
would change the baseline and invalidate stored results.

### 6. Cross-file consistency pass

Discovered and fixed 4 real consistency issues across the codebase:

#### plot.py duplicated evaluate.predict_w
`plot._eval()` and `evaluate.predict_w()` were byte-identical:
```python
def _eval/predict_w(net, device, n=1000):
    x = get_eval_grid(n=n, device=device)
    with torch.no_grad():
        y = net(x).squeeze().cpu().numpy()
    return y
```
Fix: `from evaluate import predict_w as _eval` — single source of truth, zero call-site changes.
Also dropped now-unused `torch` and `get_eval_grid` imports from plot.py.

#### plot.py had hardcoded defaults
`plot_deflection_comparison` had `n_sensors=15, noise_pct=0.10` hardcoded, which
would silently go stale if `config.py` values changed. Now imports `N_SENSORS`
and `NOISE_PCT` from config.

#### main.py unused import
Removed unused `get_eval_grid` import.

#### experiments.py internal cleanups
- Two near-identical plot functions (~45 LOC each) collapsed into shared
  `_plot_experiment()` helper.
- 6-element tuple returns with awkward `(x, means_a, means_b, phys, stds_a, stds_b)`
  ordering replaced with dict returns keyed by `x_values`, `data_means`, `data_stds`,
  `hybrid_means`, `hybrid_stds`, `physics_score`.
- Cryptic unpacking `nl, nd_m, nh_m, np_s, nd_s, nh_s = ...` replaced with
  `noise_results = run_noise_sensitivity(...)`.
- Unused `device` parameter in `_analytical_w` removed.
- Added `ValueError` for unknown mode in `_train_and_score` (was silently no-op).

**Intentionally left as-is**:
- `_Tee` class duplicated in main.py and experiments.py — identical bodies, but
  extracting a shared `utils.py` was out of scope for this pass.
- main.py remains unseeded — its output is the frozen baseline.
