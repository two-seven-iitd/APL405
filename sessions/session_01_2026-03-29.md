### 2. First run — catastrophic failure (>900% relative error)
Physics-Only and Hybrid models produced garbage. Root cause analysis:

#### Bug 1 (CRITICAL): EI value 1000x too large
- Code computed `EI = E * I = 210e9 * 8.33e-6 = 1,749,300`
- CONTEXT.md specifies `EI = 1749.3 N-m^2`
- Fix: Hardcoded `EI = 1749.3` in config.py, changed data.py to use `EI` directly

#### Bug 2: Collocation resampling every epoch
- 500 new random points each epoch = loss landscape changes every step
- Optimizer chases moving target -> wild oscillations
- Fix: Resample every 50 epochs (`RESAMPLE_EVERY = 50`)

#### Bug 3: Insufficient epochs
- Models still learning at epoch 10,000
- Fix: Increased to 20,000, later tuned to 15,000

#### Bug 4: LR scheduler too aggressive
- Patience=1000 with oscillating losses -> premature LR decay
- Fix: Increased `LR_PATIENCE` to 2000

#### Bug 5: Boundary weight too low
- `LAMBDA_GAMMA = 10.0` wasn't enough to enforce BCs
- Fix: Increased to 100.0

#### Bug 6: No gradient clipping
- Physics losses produce large gradients
- Fix: Added `torch.nn.utils.clip_grad_norm_` with `GRAD_CLIP = 1.0`

### 3. Second run — deflection sign flipped
Physics-Only and Hybrid learned POSITIVE w (upward) while analytical is NEGATIVE (downward).
Moment M(x) was perfect though.

#### Bug 7 (CRITICAL): Sign convention error in coupling equation
- CONTEXT.md says `M = -EI * w''` -> coupling loss used `(M + EI*w'')^2`
- But with CONTEXT.md's own analytical solution (w negative downward), the correct relation is `M = EI * w''`
- Verified with `tests/verify_signs.py`:
  - `M + EI*w''` residual = **1000** (WRONG)
  - `M - EI*w''` residual = **~0** (CORRECT)
- Fix: Changed `losses.py` line 39 from `M_pred + EI * w_xx` to `M_pred - EI * w_xx`
- CONTEXT.md has an internal contradiction (its coupling equation conflicts with its analytical solution)

### 4. Third run (5k epochs) — sign fixed but underfit
- Deflection direction was correct (negative/downward) but Physics-Only Rel L2 = 116%, Hybrid = 62%
- Cause: 5,000 epochs not enough for physics losses to converge
- Fix: Increased epochs to 15,000

### Beam-specific loss weights (config.py)
Loss weights are now per-beam via `LOSS_WEIGHTS` dict and `get_loss_weights(beam)`:
```
Cantilever:        lambda_omega=1.0, lambda_upsilon=1.0, lambda_gamma=100,  lambda_data=1.0
Simply Supported:  lambda_omega=1.0, lambda_upsilon=1.0, lambda_gamma=500,  lambda_data=1.0
```
- Simply supported needs higher lambda_gamma (500 vs 100) because:
  - All 4 BCs are Dirichlet (value) conditions — harder to enforce simultaneously
  - Deflection is ~10x smaller (~6.5 mm vs ~71.5 mm) so PDE residuals are smaller relative to BC violations
- BC loss functions were already separate in losses.py:
  - `loss_bc_cantilever`: w(0)=0, w'(0)=0, M(L)=0, M'(L)=0
  - `loss_bc_simply_supported`: w(0)=0, w(L)=0, M(0)=0, M(L)=0
- train.py now calls `get_loss_weights(beam)` instead of using global LAMBDA_* constants

## Coupling equation (losses.py)
```python
# Constitutive: M = EI * w''  =>  loss = (M - EI*w'')^2
return torch.mean((M_pred - EI * w_xx) ** 2)
```

### Simply Supported Beam — first run (15,000 epochs, lambda_gamma=100)
| Model        | MSE       | Rel. L2 Error | Max Abs Error | Verdict      |
|--------------|-----------|---------------|-------------|--------------|
| Data-Only    | 1.36e-08  | 2.21%         | 0.33 mm     | Excellent    |

Analysis: Results were inverted — Data-Only best, physics models worst. Cause:
- Simply-supported deflection is ~10x smaller (6.5 mm vs 71.5 mm) -> easy for Data-Only to fit
- lambda_gamma=100 was too low for 4 Dirichlet BCs -> w(0) and w(L) not enforced tightly
- Moment M(x) was still perfect for both physics models
- Fix: increased lambda_gamma to 500 for simply-supported beam

## Future scope (discussed in session)

### Hyperparameter optimization
- Current hyperparams were tuned manually (one at a time)
- Could use **Optuna** (Bayesian optimization) to search over lambda_gamma, LR, resample frequency etc.
- Loss balancing techniques (GradNorm, self-adaptive weights) are PINN-specific and more impactful than generic HPO
- Most sensitive params in order: lambda_gamma > LR/scheduler > network size > resampling

### Changing loading/boundary conditions
- Current model is trained for ONE specific setup (q=1000, cantilever, EI=1749.3)
- Changing any of these requires retraining from scratch
- **Transfer learning**: initialize from existing weights for similar problems (e.g., q=1000 -> q=1200) — converges faster
- **Parametric PINNs**: make physical parameters (q, EI, etc.) additional network inputs
  - e.g., `w_net: Input(x, q) -> Output(w)` instead of `Input(x) -> Output(w)`
  - Train over a range of parameter values, then query any value at inference without retraining
  - This is a research-active area (~2019 onwards, Raissi et al.) — methods work but best practices still evolving
  - Good "future work" section for a course report, not necessary to implement

### 7. Derived quantities — standalone plotting module (added 2026-03-30)

Created `derived_quantities/` directory — fully independent of the main project (can be deleted without affecting anything).

#### What it does
Loads saved Hybrid model weights (hybrid_w.pth, hybrid_m.pth) and generates 4 derived quantity plots:

1. **Shear Force V(x)** — line plot, analytical vs predicted
   - V(x) = dM/dx, computed via autograd from the M-net
   - Cantilever: V goes from 1000N (fixed end) to 0 (free end)
   - Simply supported: V goes from +500N to -500N (linear, zero at midspan)

2. **Max Bending Stress sigma_max(x)** — line plot
   - sigma_max(x) = M(x) * (h/2) / I, where h=100mm, I=8.33e-6 m^4
   - Shows stress at the outermost fiber

3. **Stress Contour sigma(x,y)** — 2x2 grid
   - Top-left: Analytical sigma(x,y) = M_analytical(x) * y / I
   - Top-right: Predicted sigma(x,y) from Hybrid M-net
   - Bottom-left: Absolute error
   - Bottom-right: Stress profile at critical section (x=0 for cantilever, x=L/2 for simply-supported)
   - Absolute errors ~1e-4 MPa (excellent agreement)

4. **Strain Contour epsilon(x,y)** — same 2x2 layout
   - epsilon = sigma / E

#### Cross-section assumed
- Rectangular: b = 10 mm, h = 100 mm
- I = b*h^3/12 = 8.33e-6 m^4 (matches config.py)

#### Usage
```bash
python derived_quantities/generate_plots.py --beam cantilever
python derived_quantities/generate_plots.py --beam simply_supported
```
